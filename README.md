## 🎯 О проекте

**Shopping List Manager** — это веб-приложение, которое помогает пользователям планировать покупки, избегать забытых товаров и контролировать бюджет. Приложение поддерживает совместный доступ к спискам, что удобно для семьи или коллег.

### 🔍 Проблемы, которые решает проект:
| Проблема | Решение |
|----------|---------|
| Забывание товаров в магазине | Централизованный цифровой список с синхронизацией |
| Дублирование покупок | Совместный доступ в реальном времени |
| Неудобство бумажных заметок | Адаптивный веб-интерфейс + PWA |
| Отсутствие контроля бюджета | Автоподсчёт примерной стоимости корзины |

### 👥 Целевая аудитория:
- **Пользователи** — ведут личные списки, быстро добавляют товары
- **Семьи/Друзья** — общий доступ к одному списку, синхронизация изменений
- **Администраторы** — управление пользователями и системная аналитика

## ✨ Возможности

### Для пользователя:
- ✅ Авторизация через email или Google
- ✅ Создание, редактирование и удаление списков покупок
- ✅ Добавление товаров с категорией, количеством и примерной ценой
- ✅ Отметка купленных товаров (чекбоксы/свайпы)
- ✅ Совместный доступ: приглашение по email с ролями (редактор/просмотр)
- ✅ Автоматический подсчёт общей суммы и прогресса списка
- ✅ Экспорт завершённых списков в PDF

### Для администратора:
- 🔐 Управление пользователями (блокировка, просмотр)
- 📊 Системная статистика: активность, количество списков
- 🛡️ Модерация контента и категорий

## 🗄 Модель данных

### Основные сущности:
Users (1) ←→ (N) ShoppingLists ←→ (N) Items
ShoppingLists (1) ←→ (N) ListMembers ←→ (N) Users


### Ключевые таблицы:

#### `Users` — Пользователи
| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER | Уникальный ID (PK) |
| email | VARCHAR(100) | Email (UNIQUE) |
| passwordHash | VARCHAR(255) | Хеш пароля |
| fullName | VARCHAR(100) | Имя пользователя |
| createdAt | TIMESTAMP | Дата регистрации |

#### `ShoppingLists` — Списки покупок
| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER | Уникальный ID (PK) |
| ownerId | INTEGER | ID создателя (FK → Users) |
| title | VARCHAR(100) | Название списка |
| totalBudget | DECIMAL(10,2) | Примерный бюджет |
| createdAt / updatedAt | TIMESTAMP | Даты создания/изменения |

#### `Items` — Товары
| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER | Уникальный ID (PK) |
| listId | INTEGER | ID списка (FK → ShoppingLists) |
| name | VARCHAR(150) | Название товара |
| category | VARCHAR(50) | Категория (овощи, быт. химия...) |
| quantity | DECIMAL(6,2) | Количество |
| estimatedPrice | DECIMAL(8,2) | Примерная цена |
| isBought | BOOLEAN | Статус покупки (по умолчанию `false`) |

#### `ListMembers` — Доступ к списку
| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER | Уникальный ID (PK) |
| listId, userId | INTEGER | Связка списка и пользователя |
| role | ENUM | 'owner' \| 'editor' \| 'viewer' |

> 📎 Полная ER-диаграмма доступна в папке `/docs/er-diagram.png`

## 🔌 API

### Основные эндпоинты (REST):

#### 🔐 Аутентификация
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `POST` | `/api/auth/register` | Регистрация нового пользователя |
| `POST` | `/api/auth/login` | Вход в систему, возврат JWT-токена |

#### 📋 Списки покупок
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `GET` | `/api/lists` | Получить все списки текущего пользователя |
| `POST` | `/api/lists` | Создать новый список |
| `GET` | `/api/lists/:id` | Получить детали списка с товарами |
| `PUT` | `/api/lists/:id` | Обновить название или бюджет |
| `DELETE` | `/api/lists/:id` | Удалить список |

#### 🛍️ Товары
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `POST` | `/api/lists/:listId/items` | Добавить товар в список |
| `PUT` | `/api/items/:id` | Обновить товар или отметить как купленный |
| `DELETE` | `/api/items/:id` | Удалить товар |

#### 👥 Совместный доступ
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `POST` | `/api/lists/:listId/share` | Пригласить участника по email |

> 📎 Полная спецификация с примерами запросов/ответов: [`/docs/api-specification.md`](./docs/api-specification.md)
